# ChatStory 需求规格文档

## 概述
ChatStory 是一个基于 OpenAI 兼容 API 的 Chat 对话工具，支持多种模型、写作分析、长期记忆、快捷提示词等功能。
采用单文件 HTML PWA + Node.js 服务器架构，支持本地运行和云端部署。

---

## 1. 对话 (Chat)

### 1.1 核心对话功能
| ID | 需求 | 详细说明 |
|----|------|---------|
| C1 | 发送消息 | 用户在输入框输入文本，支持 Enter 发送、Shift+Enter 换行 |
| C2 | 模型选择 | 对话界面顶部可下拉选择已获取的模型列表中的模型 |
| C3 | 获取模型列表 | 配置 API Key 后，自动从 API Base URL 获取可用模型列表 |
| C4 | API 配置 | 支持 OpenAI、DeepSeek、Moonshot、Ollama、vLLM、OpenRouter 等的 API Base URL 下拉选择 |
| C5 | 附件发送 | 发送框可增加文件附件（图片、文本等），将文件内容传给模型 |
| C6 | 流式响应 | 支持流式接收模型返回，Max Tokens 默认不限制 |
| C7 | 系统 Prompt | 支持设置系统 Prompt，发送时作为 system 消息 |
| C8 | 多轮对话 | 支持完整的多轮对话上下文 |

### 1.2 Token 统计
| ID | 需求 | 详细说明 |
|----|------|---------|
| T1 | Token 显示 | 右上角显示当前对话已用 Token / 最大上下文 Token |
| T2 | 剩余计算 | 自动计算并显示剩余可用 Token |
| T3 | 模型匹配 | 模糊匹配当前模型名称，获取对应的最大上下文 Token（如 deepseek-v4-pro = 1M） |

### 1.3 对话管理
| ID | 需求 | 详细说明 |
|----|------|---------|
| D1 | 新建对话 | 在"对话"中提供"新对话"二级功能 |
| D2 | 切换对话 | 左侧显示对话列表，可点击切换 |
| D3 | 删除对话 | 可删除历史对话 |
| D4 | 清空对话 | 可清空当前对话全部消息 |

### 1.4 消息操作
| ID | 需求 | 详细说明 |
|----|------|---------|
| M1 | 重新回答 | 模型消息下方提供"重新回答"按钮（图标），悬停显示文字 |
| M2 | 在下方重新回答 | 用户消息下方提供"在下方重新回答"按钮（图标），悬停显示文字 |
| M3 | 回答版本切换 | 重新回答后，新回复和旧回复用数字标签区分，可切换 |
| M4 | 编辑消息 | 用户和模型消息均可编辑 |
| M5 | 复制消息 | 用户和模型消息均可复制 |
| M6 | 删除消息 | 每条消息可单独删除，显示红色 ✕ 按钮 |
| M7 | 添加至长期记忆 | 每条消息气泡悬停时显示 🧠（紫色）按钮，点击弹出项目列表，将消息添加到长期记忆 |

### 1.5 右键菜单
| ID | 需求 | 详细说明 |
|----|------|---------|
| R1 | 标记到低级作家 | 选中文字后右键 → 标记到低级作家（将选中文字填入低级作家表单） |
| R2 | 标记到高级作家 | 选中文字后右键 → 标记到高级作家（将选中文字填入高级作家表单） |
| R3 | 添加至长期记忆 | 选中文字后右键 → 添加至长期记忆（弹出项目列表） |

### 1.6 页面布局
| ID | 需求 | 详细说明 |
|----|------|---------|
| L1 | 用户/模型布局 | 用户消息右侧对齐，模型消息左侧对齐，不在同一行，不强制对齐 |
| L2 | 自适应宽度 | 对话区域自适应用右侧页面，不留空白 |
| L3 | 模型回复宽度 | 模型回复最大占据整个右侧页面宽度 |
| L4 | 消息按钮 | 按钮使用透明背景图标，鼠标悬停才显示文字 |

### 测试方法 - 对话
1. **C1**: 输入文字按 Enter，验证消息发送；按 Shift+Enter，验证换行不发送
2. **C2**: 点击模型下拉，选择不同模型，验证切换成功
3. **C3**: 在设置中填写 API Key，点击获取模型，验证列表正确返回
4. **C4**: 在设置中选择不同预设（DeepSeek 等），验证 API Base URL 自动填充
5. **C5**: 点击附件按钮，选择文件，验证发送时文件内容包含在消息中
6. **C6**: 发送消息，验证模型返回完整流式响应
7. **T1-T3**: 发送多轮对话，观察右上角 Token 显示变化，验证剩余计算正确
8. **D1-D4**: 新建/切换/删除/清空对话，验证数据正确保存和恢复
9. **M1-M7**: 悬停消息验证按钮显示，点击各按钮验证功能
10. **R1-R3**: 选中文字右键，验证菜单功能和数据填入
11. **L1-L4**: 调整窗口大小，验证布局自适应和按钮显示逻辑

## 2. 快捷提示词 (Quick Prompts)

### 2.1 功能
| ID | 需求 | 详细说明 |
|----|------|---------|
| Q1 | 新建提示词 | 输入标题（可选）和内容，点击"新建并保存" |
| Q2 | 列表显示 | 已保存的提示词以列表形式显示 |
| Q3 | 使用提示词 | 点击已建好的快捷提示词，将内容贴入发送框（不自动发送） |
| Q4 | 编辑提示词 | 每条提示词旁有 ✎ 编辑按钮，可修改标题和内容 |
| Q5 | 删除提示词 | 每条提示词旁有 ✕ 删除按钮 |
| Q6 | 粘贴并发送 | 每条提示词旁有 ▶ 按钮，粘贴内容并自动发送 |

### 测试方法 - 快捷提示词
1. **Q1**: 输入标题和内容，点击新建并保存，验证列表中出现新项
2. **Q2**: 添加多条提示词，验证列表正确显示
3. **Q3**: 点击列表中的提示词，验证内容贴入发送框，不自动发送
4. **Q4**: 点击 ✎，修改内容，验证更新成功
5. **Q5**: 点击 ✕，验证提示词被删除
6. **Q6**: 点击 ▶，验证内容贴入并自动发送

## 3. 低级作家 (Low Writer)

### 3.1 功能概述
用于永久记录模型返回的文字中用户认为描写不好的地方。

### 3.2 记录管理
| ID | 需求 | 详细说明 |
|----|------|---------|
| RV1 | 提交记录 | 粘贴有问题的描写文字，选择或输入原因标签，点击"记录并分析" |
| RV2 | 记录内容 | 每条记录包含：编号、原文内容（前300字）、原因标签、日期、AI 分析总结 |
| RV3 | 标签选择 | 已有标签以下拉选择方式显示，同时显示 tag-chip（带 ✕ 删除） |
| RV4 | 新建标签 | 可输入新标签名称，点击 + 添加 |
| RV5 | 删除标签 | 每个 tag-chip 上有 ✕ 按钮，点击删除该标签 |
| RV6 | 删除记录 | 每条记录右上角有 ✕ 删除按钮，确认后删除 |
| RV7 | 记录编号 | 删除后自动重新编号 |

### 3.3 AI 分析
| ID | 需求 | 详细说明 |
|----|------|---------|
| RA1 | 自动分析 | 提交记录时自动调用 AI，根据原因分析负向规则（不应做的事） |
| RA2 | 重新分析 | 每条记录的 AI 分析旁有 🔄 按钮，点击重新调用 AI 分析 |
| RA3 | 编辑分析 | 每条记录的 AI 分析旁有 ✏️ 按钮，可手动编辑 AI 分析结果 |
| RA4 | 编辑 Prompt | "记录并分析"按钮旁有 ✏️ 按钮，可自定义 AI 分析的 Prompt |

### 3.4 负 Prompt 生成
| ID | 需求 | 详细说明 |
|----|------|---------|
| RP1 | 生成负 Prompt | 点击"负 Prompt"按钮，弹出模态窗口显示基于所有记录的负向提示词 |
| RP2 | 复制 | 模态窗口中可一键复制负 Prompt 内容 |

### 3.5 UI
| ID | 需求 | 详细说明 |
|----|------|---------|
| RU1 | 徽章 | 侧栏"低级作家"旁显示纯黑色数字（记录数量），无彩色背景 |
| RU2 | 记录列表 | 倒序显示（最新在前），条目悬停时显示操作按钮 |

### 测试方法 - 低级作家
1. **RV1**: 填写内容和原因，点击记录并分析，验证 AI 分析返回，记录出现在列表中
2. **RV2**: 验证记录包含编号、原文（截断300字）、原因、日期、AI 分析
3. **RV3**: 添加多个标签后，验证下拉和 tag-chip 都正确显示
4. **RV4**: 输入新标签名，点击 +，验证标签出现在下拉和 tag-chip 中
5. **RV5**: 点击 tag-chip 上的 ✕，验证标签被删除
6. **RV6**: 点击记录的 ✕，确认后验证记录被删除
7. **RV7**: 删除中间记录，验证编号自动重新排序
8. **RA1**: 验证 AI 分析结果不为空，显示负向规则
9. **RA2**: 点击 🔄，验证 AI 重新分析并更新结果
10. **RA3**: 点击 ✏️，修改分析内容，验证更新成功
11. **RA4**: 点击 Prompt 编辑按钮，修改 Prompt，验证后续分析使用新 Prompt
12. **RP1-RP2**: 点击负 Prompt，验证弹出模态窗口显示规则列表

| GV4 | 新建标签 | 可输入新标签名称，点击 + 添加 |
| GV5 | 删除标签 | 每个 tag-chip 上有 ✕ 按钮，点击删除该标签 |
| GV6 | 删除记录 | 每条记录右上角有 ✕ 删除按钮 |

### 4.3 AI 分析

## 5. 长期记忆 (Long-term Memory)

### 5.1 功能
| ID | 需求 | 详细说明 |
|----|------|---------|
| MM1 | 记忆项目 | 可创建多个记忆项目（如"世界观"、"角色设定"） |
| MM2 | 记忆条目 | 每个项目下可添加多条记忆，以列表形式显示 |
| MM3 | 折叠展开 | 项目可折叠/展开，箭头随状态旋转 |
| MM4 | 删除项目 | 每个项目可删除（确认弹窗） |
| MM5 | 删除条目 | 每条记忆可单独删除 |
| MM6 | 对话添加 | 可通过消息气泡 🧠 按钮或右键菜单添加至长期记忆 |
| MM7 | 注入上下文 | 发送消息时，长期记忆自动注入为 system 消息的一部分 |

### 测试方法 - 长期记忆
1. **MM1**: 输入项目名，点击新建项目，验证出现在列表中
2. **MM2**: 在项目下输入记忆内容，点击 +，验证条目显示
3. **MM3**: 点击项目标题，验证展开/折叠，箭头旋转
4. **MM4**: 点击项目 ✕，确认后验证删除
5. **MM5**: 点击条目 ✕，验证删除
6. **MM6**: 悬停消息，点击 🧠，选择项目，验证记忆添加成功
7. **MM7**: 发送对话，验证长期记忆内容出现在请求的 system prompt 中

## 6. 设置 (Settings)

### 6.1 功能
| ID | 需求 | 详细说明 |
|----|------|---------|
| S1 | 打开方式 | 点击左侧底部"⚙️ 设置"按钮，弹出模态窗口（不是侧栏展开） |
| S2 | API Base URL | 下拉预设（OpenAI、DeepSeek、Moonshot、Ollama、vLLM、OpenRouter）+ 手动输入 |
| S3 | API Key | 文本输入框 |
| S4 | 获取模型 | 点击按钮根据 API Key 和 Base URL 获取可用模型列表 |
| S5 | 模型选择 | 下拉选择模型 |
| S6 | Max Tokens | 默认不限制，可取消限制手动设置 |
| S7 | Temperature | 滑块或输入框调整 |
| S8 | 系统 Prompt | 文本域输入 |
| S9 | Gitee Token | 云端同步所需 Token |
| S10 | Gitee 仓库 | 云端同步所需仓库路径 |
| S11 | Gitee 分支 | 云端同步所需分支名 |
| S12 | 主题切换 | 深色/浅色主题 |

### 测试方法 - 设置
1. **S1**: 点击左侧底部设置按钮，验证弹出模态窗口
2. **S2**: 选择不同预设，验证 Base URL 自动填充
3. **S3-S4**: 填写 Key，点击获取模型，验证列表正确
4. **S5**: 选择模型，回到对话页验证模型切换
5. **S6**: 验证默认不限制 Max Tokens
6. **S9-S11**: 填写 Gitee 信息，点击保存，验证云端功能可用

## 7. 信息备份 (Import/Export)

### 7.1 功能
| ID | 需求 | 详细说明 |
|----|------|---------|
| B1 | 本地导出 | 导出快捷提示词、低级作家、高级作家、长期记忆为 JSON 文件 |
| B2 | 本地导入 | 从本地 JSON 文件合并恢复数据（所有类型） |
| B3 | 云端备份 | 备份至 Gitee 云端仓库 |
| B4 | 云端恢复 | 从 Gitee 云端仓库恢复数据 |
| B5 | 数据范围 | 包含：快捷提示词、低级作家（条目+原因标签）、高级作家（条目+标签）、长期记忆 |
| B6 | 合并策略 | 导入/恢复时不覆盖，去重合并 |

### 测试方法 - 信息备份
1. **B1**: 点击导出到本地，验证下载 JSON 文件
2. **B2**: 清空数据，点击从本地导入，选择之前导出的 JSON，验证数据恢复
3. **B3**: 配置 Gitee 后，点击备份至云端，验证成功提示
4. **B4**: 修改数据后，点击从云端恢复，验证数据合并
5. **B5**: 检查导出的 JSON 文件包含所有数据类型的字段
6. **B6**: 导入重复数据，验证不会产生重复条目

## 9. 服务器 (Server)

### 9.1 功能
| ID | 需求 | 详细说明 |
|----|------|---------|
| SR1 | 本地服务 | Node.js HTTP 服务器，监听 0.0.0.0:8080 |
| SR2 | 密码保护 | 访问需密码（chatstory888），登录后 session 有效 24 小时 |
| SR3 | 静态文件 | 提供 index.html、manifest.json、sw.js、图标等 PWA 文件 |
| SR4 | Git API | /api/git/status, /api/git/pull, /api/git/push, /api/git/log |
| SR5 | 数据 API | /api/data/upload（POST 保存并 git commit），/api/data/download（GET 读取） |
| SR6 | CORS | 允许跨域访问 |
| SR7 | PWA | 支持 PWA 安装（manifest.json + sw.js） |

### 测试方法 - 服务器
1. **SR1**: node server.js，验证端口 8080 监听
2. **SR2**: 首次访问需输入密码，错误密码无法访问，正确后 session 保持
3. **SR3**: 直接访问 /manifest.json 等路径，验证可访问
4. **SR4-SR5**: 调用 API 端点验证返回正确数据
