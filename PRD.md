## 1. 产品概述

### 1.1 产品定位
ChatStory 是一款面向写作者的 AI 辅助对话工具，用于与 AI 模型进行对话创作，同时提供书籍工程管理、写作分析、长期记忆等功能，帮助作者管理和分析写作内容。

### 1.2 核心用户
- 小说/故事创作者
- 需要 AI 辅助写作的用户
- 需要管理和分析写作素材的用户

### 1.3 技术架构
- **前端**: 纯 HTML/CSS/JavaScript，单页应用
- **后端**: Node.js HTTP 服务器
- **数据库**: SQLite (chatstory.db) — 所有用户数据的统一存储
- **云端同步**: Gitee API — 上传/下载数据库文件
- **AI 接口**: 兼容 OpenAI API 格式的任意模型

### 3.4 交互规范

#### 右键菜单
- 选中对话文本后右键弹出
- 菜单项: 🏴 标记到低级作家、🏆 标记到高级作家、📋 复制、📄 全选、🧠 添加至长期记忆
- 点击菜单项后菜单自动关闭
- 点击页面其他区域自动关闭

- 从右键菜单或"＋ 新增分析"按钮打开

- 支持"正Prompt"/"负Prompt"生成

- 每个子模块可折叠/展开
- 记录以紧凑格式显示: #编号 + 内容摘要 + 删除按钮

### 3.5 移动端适配
- 侧边栏默认隐藏，通过汉堡菜单按钮滑出

## 5. 页面流程

### 5.1 用户首次使用
1. 访问 http://localhost:8080/
2. 输入密码 chatstory888 登录
3. 进入主界面，左侧侧边栏展开
4. 点击"书籍工程" → 创建工程
5. 点击"对话" → 新对话 → 输入消息
6. 选中消息文本 → 右键 → 标记到作家 → 分析弹窗

### 5.2 数据持久化流程
1. 用户操作 → save() 被调用
2. 立即写入 localStorage (快速响应)
3. 500ms 防抖后 POST 到 /api/data/save
4. 服务器写入 SQLite 数据库
5. 下次页面加载 → load() 从服务器获取最新数据 → 合并到本地

### 5.3 云端同步流程
1. **上传**: 点击备份 → 上传数据库 → 调用 Gitee API → 上传 chatstory.db 的 base64 编码
2. **下载**: 点击恢复 → 从 Gitee 下载 chatstory.db → 替换本地数据库 → 刷新页面

## 6. API 接口文档

### 6.1 服务器 API

| 端点 | 方法 | 说明 | 请求体 |
|------|------|------|--------|
| /login | POST | 用户登录 | pwd=chatstory888 |
| /api/data/load | GET | 加载全部数据 | - |

| /api/data/info | GET | 数据库信息 | - |
| /api/sync/upload | POST | 上传到 Gitee | {token, repo, branch} |
| /api/sync/download | POST | 从 Gitee 下载 | {token, repo, branch} |
| /api/git/pull | POST | Git 拉取 | {token, repo, branch} |
| /api/git/push | POST | Git 推送 | {token, repo, branch, message} |
| /api/git/log | GET | Git 日志 | - |
| /api/data/upload | POST | 遗留: 上传数据 | JSON |
| /api/data/download | GET | 遗留: 下载数据 | - |
| /api/github/push | POST | GitHub 推送 | {token, repo, branch, message} |
